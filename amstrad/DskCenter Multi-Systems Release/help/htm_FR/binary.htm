Outils Binaires ...

Cet outil permet la modification d'une image de disquette grâce à des outils de base Couper, superposer, allonger ou rétrécir.

Le « File trimmer » :


Il permet de couper un « Header » (ou l'entête d'un fichier) dans un format différent à partir d'un index de décalage.

    Exemple: Pour la conversion de fichiers ATR en XFD, il suffit de déterminer une entête de 16 octets sur le format ATR, et de sauvegarder le fichier obtenu en XFD.

Le « Boot patcher » :

Il permet de superposer un fichier Boot ou entête sur une disquette.

    Exemple: Je veux créer une disquette Cpm avec un fichier boot d'une autre disquette ou changer l'entête du Média.

Le « Raw format converter » :


Il Permet de changer le format d'une disquette image en mode native pour la convertir dans un autre format

Pour réduire ou augmenter la taille d'une disquette.

  • Réduire la taille d'une image: le fichier est tronqué aux dimensions du support choisit (les données en trop seront supprimées du fichier sauvegardé).

    Exemple: Je veux récupérer une disquette de 720ko au lieu de 1440ko ou bien récupérer une disquette de 5 Mo après être passé dans un émulateur ( sur CPC ce n'est pas rare ) .

    Nota: Sous Dos, pour récupérer un média stable, il faut « patcher » un nouveau « boot-strap », une nouvelle entête de disque.

  • Pour augmenter la taille d'une disquette d'une disquette.

    Augmenter la taille d'une image : le fichier est augmenté d'autant avec un caractère défini. Pour Passer d'un support physique à un autre.

    Exemples: Je veux augmenter la taille d'une disquette système de 720ko au lieu de 1440ko pour y placer un logiciel à démarrage automatique. (mêmes précaution avec le boot-strap Ms-DOS) ou l'image disque que j'ai récupéré possède des erreurs de lecture et ne fait que 650ko sur les 720ko! La taille correcte peut être retrouvée ou une image CopyQM n'est pas assez grande! (information du format dans l'entête du fichier)

    Pourquoi en mode Natif [IMG] ?

    En fait, pour une raison simple : les formats à structure propriétaire possédent des informations sur la structure proprement dite : le nombre de secteurs, de pistes, sur la capacité de clusters ... et peuvent même posséder des informations sur chaque secteur!

    Dans ce cas, le fichier sera rejeter par le logiciel hôte, même s'il supporte la taille du format donné.

    Une conversion à l'aide d'un utilitaire spécial sera donc utile pour ressortir les pistes une à une.
